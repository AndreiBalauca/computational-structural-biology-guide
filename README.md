# Computational Structural Biology Guide

A beginner-friendly guide to computational structural biology and drug discovery workflows, including structure retrieval, molecular visualisation, docking, molecular dynamics, AI-predicted structures, and reproducible analysis.

The aim of this repository is to document the broader workflow used in computational structural biology, from choosing a protein target to preparing structures, running computational methods, analysing results, and interpreting outputs carefully.

This repository is separate from my AMBER-specific notes. It focuses on the full research pipeline rather than one software package.

---

## Workflow Overview

```text
Protein target
      ↓
Structure search
      ↓
Structure quality assessment
      ↓
Molecular visualisation
      ↓
Protein and ligand preparation
      ↓
Docking / molecular dynamics / AI modelling
      ↓
Analysis and interpretation
      ↓
Reproducible documentation
```

## Purpose

This repository is designed to organise the main concepts and practical steps involved in computational structural biology workflows.

It focuses on:

- choosing suitable protein structures
- understanding experimental and predicted structures
- checking structure quality before computation
- preparing proteins and ligands
- understanding docking and molecular dynamics workflows
- comparing AMBER and GROMACS at a beginner level
- using AI-predicted structures cautiously
- interpreting computational results without overclaiming
- documenting workflows so they are reproducible

## Research Data Disclaimer

This repository contains educational notes, generic workflows, and public-learning material only.

It does **not** contain unpublished research data, confidential structures, lab-owned workflows, trajectories, docking results, or project-specific analysis from any research group.
